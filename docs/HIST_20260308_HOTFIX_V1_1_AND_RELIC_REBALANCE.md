## 📖 一、 概述与核心目的

本文件作为《余烬回响 (Embers' Echoes)》项目的历史性归档文档，记录了 **2026-03-05** 至 **2026-03-08** 期间进行的高能级数值热更新 (Hotfix V1.1) 与终局文明遗物合成成本的大规模重组工作。

在重构前，游戏处于极其脆弱的发展阶段，存在若干致命的逻辑和数值错配，直接导致体验断层和逻辑死锁。本阶段核心解决的问题包括：

1. **数值永久死锁解决**：此前文明遗物的合成材料成本被设定在极其高昂的范围（如数千余烬、20个浓缩液），而游戏满级建筑提供的硬上限（余烬仓容 600，浓缩液仓容 10）根本无法容纳这些消耗。这导致在不修改代码的情况下，玩家理论上永远不可能达成真结局，形成致命死锁。
2. **理智上限（SAN）动态解耦**：修复了“认知屏障”建筑仅有描述文案、而没有实质性增加最大 SAN 值的逻辑漏洞。通过解耦 `$SM` 的数据层，将理智限制在动态读取的上限之内，确保了高阶同化玩法的实现。
3. **低语容量管理与溢出警报**：重构了数据金库与仓容，针对极易溢出的低语值增加了状态满额通知，大幅优化了玩家在盲盒式探索中的感知体验。
4. **高阶碎片智能免重掉落 (Smart Loot & Bonus Loot)**：原机制中副本掉落可能重复，导致全收集遥遥无期。本阶段全面推行“智能免重判定”，并针对已经达成全集齐的骨灰级玩家提供了“溢出奖励补偿机制”，保证了后期探索的持续正反馈。
5. **历史冗余资源物理精简**：物理删除 ADR 继承下来的 `img/` 遗留资源，释放开发和版本控制开销，防止大文件混淆。

## 🛠️ 三、 Hotfix V1.1 核心功能重组明细

> [!NOTE]
> Hotfix V1.1 的首要任务是完成底层状态数据与上限的解耦，为高阶副本的进出提供稳健的安全网。

### 1. 理智上限（SAN）动态扩展与 Clamp
*   **痛点**：此前的【认知屏障】建筑在构筑完成时仅仅弹出通知，底层的最大理智并没有上升，导致玩家获得的理智奖励在 100 上限处被无情折损。
*   **动态扩展逻辑**：在 [sanity.js](file:///Users/quantumrose/Documents/Emberois/embers-echoes/script/sanity.js#L280-L284) 中废除写死限制，重构了 `getMaxSan` 方法，从全局状态中实时计算值：
    $$\Large \text{maxSan} = 100 + (\text{buildings.cognitiveBarrier} \times 10)$$
*   **全局 Clamp 封顶**：在 [state_manager.js](file:///Users/quantumrose/Documents/Emberois/embers-echoes/script/state_manager.js#L148-L151) 中替换全局理智更新代码，凡是增加或恢复理智的行为，统一通过 `Math.min(currentSan + amount, $SM.get('character.maxSan'))`（内部统一读取自 `Sanity.getMaxSan()`）进行封顶：
    ```javascript
    var maxSan = (typeof Sanity !== 'undefined' && Sanity.getMaxSan) ? Sanity.getMaxSan() : 100;
    newVal = Math.max(0, Math.min(maxSan, newVal));
    ```

### 2. 副本高阶碎片智能免重掉落（Smart Loot）
*   **痛点**：深渊遗迹副本结算掉落是全结局遗物的唯一产出路径。若允许重复掉落，概率折损将引发挫败感。
*   **免重池过滤器**：在 [combat.js](file:///Users/quantumrose/Documents/Emberois/embers-echoes/script/combat.js#L260-L279) 中新增 `missing` 过滤器，确保新掉落的碎片必然属于玩家“未拥有且尚未用其合成出对应遗物”的池子：
    ```javascript
    var HIGH_TIER_FRAGS = ['frag_turing', 'frag_klein', 'frag_biotech', 'frag_scroll', 'frag_watch', 'frag_recorder'];
    var missing = HIGH_TIER_FRAGS.filter(function (id) {
        var recipe = Narrative.dict.craftingRecipes && Narrative.dict.craftingRecipes.find(function (r) {
            return r.fragments && r.fragments.indexOf(id) !== -1;
        });
        var relicCrafted = recipe ? $SM.hasRelic(recipe.relicId) : false;
        return !$SM.hasFragment(id) && !relicCrafted;
    });
    ```
*   **全收集代偿机制**：若 `missing` 长度为 0（证明玩家已是全集齐状态），系统在保留掉落普通碎片的同时，动态将 `Combat.Dungeon.bonusLoot` 置为 `true`。在副本彻底结算时（[combat.js:L331-L336](file:///Users/quantumrose/Documents/Emberois/embers-echoes/script/combat.js#L331-L336)）给予 **+100 异常样本** 与 **+10 低语值** 的高额观测补偿，形成正向资产流转。
